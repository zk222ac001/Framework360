# 01 - System Overview

## Project overview

Framework360 is a compliance management web application. It helps a company understand:

- Which compliance frameworks apply to the company
- What requirements the company must follow
- Where the company has compliance gaps
- What actions are needed to close those gaps
- What evidence is available to prove compliance
- Which systems, vendors, business processes, and dependencies are connected to compliance work

In simple words, Framework360 helps a company move from unclear compliance work to an organized overview of requirements, risks, tasks, and evidence.

---

## Main technical parts

The application has three main technical parts:

```text
Frontend  -> What the user sees in the browser
Backend   -> The API/server that handles business logic
Database  -> Where users, companies, frameworks, evidence, tasks, and other data are stored
```

The project uses:

```text
React + TypeScript  -> Frontend
Node.js + Express   -> Backend
PostgreSQL + Prisma -> Database
Docker Compose      -> Running the full system locally
```

## What problem does Framework360 solve?

Companies often need to follow different compliance frameworks such as GDPR, ISO 27001, or NIS2. Without a system, this can become confusing because requirements, documents, evidence, systems, vendors, and tasks may be spread across emails, spreadsheets, and folders.

Framework360 currently defines 19 compliance frameworks in the database schema:

2. NIS2

### Supported framework descriptions

| Framework | Description |
|---|---|
| GDPR | General Data Protection Regulation. Helps organizations protect personal data, manage privacy rights, document processing activities, and prove lawful handling of personal information. |
| NIS2 | EU cybersecurity directive. Focuses on cybersecurity risk management, incident reporting, supply chain security, and resilience for important and essential organizations. |
| DORA | Digital Operational Resilience Act. Applies mainly to financial entities and focuses on ICT risk management, incident reporting, resilience testing, and third-party technology risk. |
| AI_ACT | EU Artificial Intelligence Act. Helps organizations manage risks from AI systems, including governance, transparency, documentation, human oversight, and responsible AI use. |
| CRA | Cyber Resilience Act. Focuses on cybersecurity requirements for products with digital elements, including secure design, vulnerability handling, and lifecycle security. |
| DATA_ACT | EU Data Act. Covers access to and use of data, especially data generated by connected products and services, with rules for sharing and portability. |
| EIDAS | Electronic Identification, Authentication and Trust Services regulation. Covers electronic signatures, digital identity, trust services, and secure electronic transactions. |
| CER | Critical Entities Resilience directive. Focuses on resilience of critical entities that provide essential services, including risk assessment, continuity, and incident handling. |
| ISO27001 | International standard for Information Security Management Systems. Helps organizations manage information security risks using policies, controls, risk assessments, and continuous improvement. |
| ISO27002 | Security control guidance standard. Provides best-practice controls for information security, such as access control, asset management, operations security, and supplier security. |
| ISO27701 | Privacy Information Management extension to ISO 27001. Helps organizations manage privacy controls and personal data protection responsibilities. |
| ISO22301 | Business Continuity Management standard. Helps organizations prepare for disruptions, maintain critical operations, and recover from incidents. |
| ISO42001 | Artificial Intelligence Management System standard. Helps organizations govern, manage, and control AI systems responsibly and consistently. |
| SOC2 | Service Organization Control 2 framework. Used mainly by service providers to prove controls related to security, availability, processing integrity, confidentiality, and privacy. |
| CIS_CONTROLS | Center for Internet Security Controls. A practical set of prioritized cybersecurity safeguards for reducing common cyber risks. |
| NIST_CSF | NIST Cybersecurity Framework. Helps organizations structure cybersecurity work around identify, protect, detect, respond, and recover activities. |
| PCI_DSS | Payment Card Industry Data Security Standard. Applies to organizations that store, process, or transmit payment card data and focuses on protecting cardholder data. |
| TISAX | Trusted Information Security Assessment Exchange. Used mainly in the automotive industry to assess and exchange information security maturity with partners. |
| D_MAERKET | Danish D-seal / D-mærket. A Danish trust mark focused on responsible digital behavior, data protection, IT security, and digital accountability. |

Framework360 connects everything in one place:

```text
Framework -> Requirement -> Gap -> Action -> Evidence -> Approval -> Compliance progress
```

Example:

```text
Framework: ISO 27001
Requirement: Maintain an asset inventory
Gap: No updated asset inventory exists
Action: Create updated asset inventory
Evidence: Asset_Inventory.xlsx
Status: In progress or completed
```

## Simple summary

Framework360 turns compliance confusion into organized work.

Instead of having documents, vendors, systems, and tasks scattered everywhere, the system connects them together:

```text
Company -> Frameworks -> Requirements -> Gaps -> Actions -> Evidence -> Audit readiness
```
